Welcome to CA Application Performance Management community

This page describes how you can contribute to the CA Application Performance Management (APM) community and steps to follow to release your community contribution.

Learn about the CA APM Developer Community

Before you begin, check out the CA APM Developer Community, where you can learn about different projects, collaborate with other contributors, find how-to guides, and hang out with really cool people.

Join the CA-APM organization

Create a github account, and request to join the organization by sending an email to team-apm-marketplace.pdl@broadcom.com.

Fork a repo or generate a pull request

If you want to contribute to an existing repository, simply fork the repository, make your changes and send a pull request. If you need help on this, you can contact for example the person who has done the latest commits to that repository.

License to be used

Release your contribution

Steps for submitting pull requests and releasing your work are outlined in the CONTRIBUTING.md file in each repository. You can also visit the APM Dev community for more details and instructions.
